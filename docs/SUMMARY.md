# Summary
> **Review (2025-10-11):** Captured the wallet TLS client migration, updated dependency guidance after retiring `native-tls`, and reaffirmed the dependency-sovereignty pivot checkpoints. `contract tls status` now renders human-readable or JSON retention reports (fed by the new `tls_env_warning_*` gauges, fingerprint counters, unique-fingerprint gauges, and backed by the `TlsEnvWarningSnapshotsStale` alert) so operators and automation surface stale prefixes without bespoke glue, `contract telemetry tls-warnings` mirrors local node snapshots with optional JSON/label filters, per-fingerprint totals, and `--probe-detail`/`--probe-variables` helpers, `/export/all` bundles `tls_warnings/latest.json`/`status.json` for offline review, and `tls-manifest-guard` accepts quoted env exports after the latest hardening.
> Dependency pivot status: Runtime, transport, overlay, storage_engine, coding, crypto_suite, codec, serialization, SQLite, TUI, and TLS facades are live with governance overrides enforced (2025-10-11).
